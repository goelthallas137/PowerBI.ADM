# 5. Dashboard e publicação

## Escolha de visual por tipo de pergunta

| Pergunta | Visual sugerido |
|---|---|
| Evolução ao longo do tempo | Gráfico de linha |
| Comparação entre categorias (departamentos, cargos) | Gráfico de barras |
| Composição de um total | Gráfico de pizza/rosca (poucas categorias) ou barras empilhadas |
| Detalhe linha a linha | Tabela/matriz |
| Números-chave em destaque | Cartões (KPI cards) |

## Boas práticas rápidas

- No máximo 5–7 visuais por página — excesso confunde mais do que informa
- Um filtro (slicer) de período visível em todas as páginas
- Nomear medidas e visuais com linguagem de negócio, não com nomes técnicos de coluna

## Publicação

1. Salvar o `.pbix` localmente
2. Publicar para o Power BI Service (workspace da equipe)
3. Definir quem pode ver/editar (permissões do workspace)
4. Configurar atualização automática se a fonte de dados mudar com frequência
